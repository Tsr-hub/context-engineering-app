# マイクロブログ 要件定義

## 概要

個人用マイクロブログ。Markdownファイルをコンテンツソースとし、VercelにデプロイするNext.js製Webアプリ。

---

## 機能要件

### 閲覧機能
- 投稿一覧ページ（新着順）
- 投稿詳細ページ
- Markdown形式のレンダリング（見出し・リスト・コードブロック等）
- 誰でも閲覧可能（認証なし）

### 投稿管理
- 投稿はローカルでMarkdownファイル（`.md`）を作成・編集
- `git push` によりVercelが自動ビルド・デプロイ
- 管理画面・ログイン機能なし

---

## 非機能要件

### 技術スタック
- **フレームワーク**: Next.js 15（App Router）
- **UIライブラリ**: shadcn/ui
- **コンテンツ**: Markdownファイル（ローカルファイルシステム）
- **デプロイ**: Vercel（静的生成 / SSG）

### コンテンツ形式
- テキストのみ（画像・動画は対象外）
- Markdown記法対応
- フロントマター（frontmatter）でメタデータ管理
  - `title`: タイトル
  - `date`: 投稿日時
  - `slug`: URL識別子

### UI/UX
- shadcn/ui を用いたスタイリッシュなデザイン
- レスポンシブ対応

---

## ディレクトリ構成（案）

```
/
├── posts/              # Markdownファイル置き場
│   └── YYYYMMDD-slug.md
├── app/
│   ├── page.tsx        # 投稿一覧
│   └── posts/[slug]/
│       └── page.tsx    # 投稿詳細
└── lib/
    └── posts.ts        # Markdownパース・取得ロジック
```

---

## 投稿ワークフロー

1. `posts/` 配下に `.md` ファイルを作成
2. フロントマターにタイトル・日付・slugを記載
3. `git push` → Vercelが自動ビルド・公開

---

## スコープ外（今回対象外）

- 画像アップロード
- コメント・いいね
- タグ・カテゴリ
- 検索
- RSSフィード
- 認証・ログイン
